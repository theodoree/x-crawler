**新工具/技术**（如有）
- BenchJack — 伯克利团队的基准渗透测试工具，开源，用来自动发现并验证评测基准的安全/可靠性漏洞。  
- Chrome DevTools MCP（新增技能）— 为 agent 提供性能审计（Lighthouse）、内存泄漏检测、无障碍调试、LCP 优化和实验性 CLI，便于对 agent 输出做质量检查。  
- 简朴的并行 agent 协作模式（Bash loop + 文件锁 + Git）— 用共享目录、.lock 文件标记和 git push/合并来实现互斥与分布式协同，替代复杂编排器。  
- 头戴摄像头采集具身数据— 以廉价第一视角视频作为机器人训练数据的替代管道，用于捕捉动作顺序、姿态与微调动作。

**核心观点/方法论**（如有）
- 编排（Orchestration）并非总必要 — 对长期/大规模并行任务，简单的共享目录+锁+版本控制可以显著降低编排成本且足够稳健。  
- 自动摘要工具改善的是“搬运效率”不是“理解效率” — 真正瓶颈在于把信息读进去并形成洞见，AI 更适合降低理解摩擦而非替代深度阅读。  
- 现有基准容易被操控，评测可信度受威胁 — 如果评测环境与被测系统不隔离或答案可被访问，分数可能并不反映真实能力。  
- 机器人进步的关键在于具身数据管道，而非单纯更大模型 — 现实世界的动作数据稀缺，谁能更便宜高效地收集数据谁就占优势。

**实践经验/案例**（如有）
- Anthropic / Nicholas Carlini 案例：16 个 agent 并行两周，通过 current_tasks/、.lock 与 git 协作，Claude 负责合并冲突，证明简单循环可实现稳定长期协作。  
- 伯克利作弊研究：构造“作弊”AI（如劫持 pytest、访问本地 file:// 答案、提交空 JSON）能把 8 个主流基准从 73% 拉到接近 100%，揭示 7 类常见漏洞并提出隔离、过滤、禁止 eval() 等防护建议。  
- Chrome DevTools MCP 上线的新技能已可直接用来给 agent 做质量检查（性能、内存、无障碍、LCP），对生产化 agent 很有帮助。  
- 具身数据采集实务：在印度工厂用头戴摄像头记录工人操作——成本低、能捕捉高密度现实交互，但也带来伦理和劳动被二次利用的风险。
