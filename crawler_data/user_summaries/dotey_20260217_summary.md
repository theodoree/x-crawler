**新工具/技术**
- Claude Sonnet 4.6：Anthropic 发布的 Sonnet 系列重大升级，编程、智能体规划、长上下文推理与计算机操作能力大幅提升，价格仍维持 Sonnet 级别。
- 1M token 上下文窗口（beta）：可容纳整库代码或多篇长文，模型在如此长上下文中的有效推理能力是关键改进点。
- Claude in Excel + MCP 连接器：可在 Excel 内直接调用金融/数据源（S&P、PitchBook、Moody's 等），降低跨工具整合成本。

**核心观点/方法论**
- 平台收编/架构税论：招揽关键架构作者常被解读为掌握底层协议与“收税权”的手段，意味着谁掌控交互协议层就能决定商业规则，需警惕平台化风险。
- 招聘-理解-重塑-变现流程：作者视角是：公开招聘是头条，真正目的是把架构拿进自家围墙并实现变现。
- AI 文风模板化警示：多模型在输出上出现相似“AI式”写作模板（诸如“不要”“一句话总结”等），说明提示工程和模型风格趋同，影响可读性与差异化。

**实践经验/案例**
- 编程能力实测：内部/用户偏好数据显示 Sonnet 4.6 在代码改写、多步任务上更稳、重复逻辑合并更好（用户偏好 ~70% vs 4.5，59% 在某情境下优于 Opus）。
- 计算机操作基准进步：OSWorld 分数从 14.9% 提升到 72.5%，表明实际软件环境自动化能力已从实验性走向实用。
- 长上下文与策略表现：100万 token 上下文使得复杂长期规划与模拟经营（Vending-Bench）表现出更成熟的策略切换能力。
- 安全改进：对提示词注入的抵抗力提升，降低网页/工具被劫持的风险，但仍需持续监测。
- 产品选型建议：大多数场景推荐 Sonnet 4.6（性价比高）；需要极致深度推理/零容错的任务仍倾向 Opus。
- 其他观察：转发的麻将外挂与“供参考”链接显示作者同时关注落地工程与社区讨论，提示筛选信息与实测验证的重要性。
