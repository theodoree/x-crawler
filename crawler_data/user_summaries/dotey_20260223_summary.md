**新工具/技术**（如有）
- OpenClaw — 一种以 Agent/工作流为中心的自动化工具，能直接操作邮箱等服务；暴露出“上下文压缩（context compaction）”导致指令丢失的风险。  
- Claude Code（Opus 4.6）— 用于编码类 Agent 的模型/平台，适合做自动化编程和 Coding Agent。  
- baoyu-skills（含 baoyu-cover-image、baoyu-image-gen）— 开源的 Skill 集合，用于封面图和图像生成的任务编排。  
- Gemini API + Nano banana pro — 用作图像生成的后端/模型接口，配合 Skills 产出高质量封面。  
- Anthropic 的 COBOL 自动化工具 — 面向传统企业遗留系统（COBOL）的自动化解决方案，显示 AI 对传统 IT 的冲击。  
- 蒸馏攻击（distillation attacks）— 规模化用强模型输出训练自家模型的技术/攻击手段，Anthropic 报告把它当作未经授权的数据抽取风险。

**核心观点/方法论**（如有）
- 建造变便宜，PM 的价值来源要从“写需求/控制开发”转向“判断做什么最有价值”。重要性：决定方向的能力在低成本构建时代更稀缺。  
- 不要把核心护城河押在补模型能力的技巧上（如临时 prompt 补丁），因为模型能力在快速被抹平。重要性：依赖短期技术债容易被淘汰。  
- 深层的 Prompt Engineering（理解模型认知、把复杂问题分解给模型、设计人机协作工作流）会越来越重要，而非记住几点模板。重要性：能放大模型能力并保持差异化。  
- 更新心智模型比被动认错更关键——要主动感知并重构假设。重要性：在指数变化环境中速度是竞争力。  
- 复杂系统编排+领域知识不可替代：组织阻力、权限和数据孤岛不是纯技术问题，编排需要专业判断。重要性：只有结合领域理解的编排才能落地产出价值。  
- 对于工程师，稀缺变成主动性（Agency）、品味（Taste）和质量（Quality）。重要性：当 AI 把“80分工作”普及，差异来自最后一段打磨与判断。

**实践经验/案例**（如有）
- OpenClaw 邮箱误删事件：在真实大规模上下文下“上下文压缩”丢失关键指令，导致自动化执行严重错误，强调在真实环境要做更严格的边界/确认机制。  
- Anthropic 报告的蒸馏攻击：三家实验室用 2.4 万个虚假账号、1600 万次交互提取 Claude 能力，提示模型服务面临被规模化抽取和滥用的合规与安全风险。  
- 封面图实操流水线：dotey 用 Claude Code + baoyu-skills + Gemini API（Nano banana pro）生成卡通化封面，证明用 Agent+Skill 编排可以高效产出视觉资产。  
- 对学习/求职的实操建议：做完整的项目并公开展示（哪怕粗糙），用项目体现主动性、品味和质量，比单纯学工具更能获得机会。  
- 学习策略建议：对工具层保持 20% 的感知，80% 精力深耕行业认知和用户理解，以应对工具快速迭代。
