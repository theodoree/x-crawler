**新工具/技术**（如有）
- Claude Code（CLI Agent 工具）：以命令行为中心的开发工具包，专注于与 Claude 模型深度配合，便于嵌入 CI/CD、远程控制和自动化工作流。  
- Claude Code 远程控制：新的特性允许在本机运行会话并用手机/其它设备接管，适合长任务或移动中监控。  

**核心观点/方法论**（如有）
- AI 资本支出并不等同于 GDP 增长（高盛/Jan Hatzius）：大量高端芯片和硬件为进口，按 GDP 核算会被抵消，因而巨额 AI 投入对 2025 年的美国 GDP 贡献“基本为零”。重要性：提醒不要把资本开支的口号化叙事直接等同于宏观增长。  
- 企业使用 AI 未必带来短期生产力提升（NBER 调查）：70% 已用 AI，但约 80% 企业看不到就业或生产力改善，提示落地效果与预期存在差距。  
- 规范驱动开发（SDD）的问题：静态规范易过时，会误导人或 Agent；应把“规范”设计为人机共治、可随执行动态更新的活文档。重要性：减少文档-实现不同步带来的风险。  
- 迭代与最小可运行版本：用短周期、可运行的最小版本控制风险并验证假设，保证 AI 生成代码不“失控”。  
- Agent-first 开发范式：以 Agent 为中心的 CLI 工具更利于聚焦任务上下文、移植性和嵌入式自动化；IDE 集成虽友好但常带入噪音与上下文膨胀。  
- 数据飞轮与闭环优势：自家模型+专属工具能形成训练-产品-数据的正反馈循环，第三方工具难以做到同等优化与长期护城河。  
- 对 Agent 自动更新 Spec 的保留意见（dotey）：自动同步会消耗 Token、带来一致性问题，且很多任务不需要全局理解；需权衡成本与必要性。  

**实践经验/案例**（如有）
- Antigravity 封禁事件：大量用户把后端当代理通道使用，超过 90% 流量并非正常功能调用，公司为保护服务质量紧急封号并提供恢复路径。经验：要有监测策略、明确 ToS 和快速恢复流程以兼顾安全与用户体验。  
- Claude Code 在工程实践中的落地：可用于自动代码审查、自动修复、在 PR/CI 流程触发 Agent 操作，且已被扩展到批量文件处理、报告生成等非编程场景。实践价值：把 Agent 当“执行者”嵌入工作流，人的角色转为指挥与审查。  
- 协作式 Spec 工作流示例（Augment Code 思路）：由人设目标、Agent 草拟并在执行中同步变更，人审阅批准后继续，像“优秀的初级工程师”那样反馈关键信号，能保持规范与实现一致而不需要人工持续维护。  
- 教育与能力构建（斯坦福 CS146S）：高校课程已开始教“规划→AI生成→修改→循环”的现代开发流程，说明产业界和学术界都在把这一范式制度化。
