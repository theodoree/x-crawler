**新工具/技术**  
- Cloudflare 的 Markdown 内容协商：网页能根据 content-negotiation 返回 Markdown 给 Agent，让自动化抓取/解析更可靠而非解析 HTML。  
- WebMCP（Chrome 提早预览）：为网站和 AI Agent 提供标准化接口（声明式表单 + 命令式 JS），让站点主动暴露可被调用的工具集。  
- Gemini Skill / Google Skills 生态：把 API/能力打包成可注入 Agent 的 Skill，降低某些集成成本但对低频用户价值有限。  
- Agent-first 页面输出（Markdown/CLI/MCP）：趋势是为 Agent 优化返回格式与接口，减少对 DOM 爬取的脆弱依赖。  

**核心观点/方法论**  
- Skills 工作流 vs 传统工作流：传统是人编排、代码驱动且遇错即报错；Skills/Agent 工作流让 Agent 自行尝试/修复并迭代 skills，提升鲁棒性与自主性。重要性在于把“运行时决策”从静态代码转给能反思的 Agent。  
- Agentic Engineering（意图驱动逻辑）：用语义分支取代刚性 if/else，动态工具链即时补短板，内置反思/闭环保证任务持续推进，这是下一代软件的核心循环。  
- 把 Agent 视为一等公民：需要标准（如 WebMCP）与站点声明式接口，才能实现更快、更准、更安全的自动化交互。  
- 基础设施与组织能力决定效果：像 Spotify 的“没写一行代码”更多是工具+成熟 CI/CD 与审查流程下的角色转变，而不是任意公司可直接复制的魔法。  
- 警惕话语与估值驱动的叙事：高管/媒体为投资者/流量美化成果，真实 ROI 与部署复杂度需谨慎评估。  

**实践经验/案例**  
- Spotify/Claude 场景：资深工程师转为“生成并审查”代码是事实，但依赖多年投入的自动化测试、Backstage、代码所有权等基础设施。  
- WebMCP 早期预览：适合想让 Agent 执行复杂站内操作的产品（客服表单、购物流程、订票等），开发者可申请试用并提前设计声明式接口。  
- Gemini Skill 实践观察：对频繁调用或标准化流程有帮助，但对偶发用户价值有限，文档复制有时更高效。  
- Prompt 拼接/模式差异导致的 Bug：工具链细节（如换行拼接规则）会引发意外行为，提醒在 Agent/Skill 集成时要严格对齐提示工程与边界条件。
