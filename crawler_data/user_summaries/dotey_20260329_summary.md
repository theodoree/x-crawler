**新工具/技术**（如有）
- Pretext（Cheng Lou）— 纯 TypeScript 的文本测量/排版库，脱离 DOM 测量、避免 reflow，基准速度可达传统方法的数百倍，支持多语言混排和 Emoji。  
- yetone 开源的语音输入法/Prompt（语音输入相关源码）— 开放源码，便于社区复用和验证构建过程。  
- AI 助力测试（Claude Code / Codex 在 Pretext 开发中用于大量自动化测量校准）— 用 LLM/代码模型做大规模回归验证与精度打磨。

**核心观点/方法论**（如有）
- 用“组合替代穷举 / 正交分解”来做风格定制——把“风格”拆成若干独立维度，少量选项的排列组合能覆盖大量变体，既降低维护成本又保留个性化。重要性：可扩展地实现高度定制化且降低复杂度。  
- 人机协同选择优于全自动——在多维选项中让用户参与选择（或由 AI 建议并由用户确认），能更精准表达意图并避免 AI 的“趋向平均”问题。  
- 把文本测量从 DOM 中剥离是底层级优化——将测量逻辑上移可以消除昂贵的浏览器回流，解锁高性能、实时和大规模文本渲染场景（游戏、虚拟滚动、多栏排版等）。

**实践经验/案例**（如有）
- 大规模竞品测试（macOS 语音输入）——通过安装多家商业/开源产品、大量中英语音录入进行对比，既提升了写作效率，也给作者提供了反馈，用绿/黄/红点做快速推荐。  
- Pretext 的实战 demo：数十万不同高度文本框的虚拟滚动（120fps）、响应式多栏杂志实时重排、聊天气泡自动贴合、用可变宽度字体做 ASCII 艺术，甚至跑小游戏（DOOM、放烟花、Bad Apple）——说明脱 DOM 测量带来的场景价值。  
- 开发流程值得借鉴：用自动化工具和 LLM 持续测量并迭代数周，体现了 AI 在工程化精度打磨中的作用。  
- 社区与开源效果：yetone 开源后社区能重用/验证源码并录制构建过程，利于信任建立与二次开发。  
- 建议谨慎：部分同行指出 Pretext 并不能完全替代 CSS（目前约覆盖部分排版能力），宣传时要避免过度宣称其取代性。
