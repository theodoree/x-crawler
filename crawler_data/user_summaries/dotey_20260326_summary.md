**新工具/技术**（如有）
- Type4Me — macOS 语音输入本地/云双引擎 + 大模型后处理，支持命令模式和本地存储，强调数据可控与可扩展的插件架构。
- bun（配合 npx -y bun）— 用于直接执行 TypeScript 脚本，减少编译步骤，使 Agent 能够快速定位并修补运行时代码问题。
- SGO（语义梯度优化引擎）— 开源的 AI 模拟引擎，用合成、人口统计对齐的用户反馈快速做产品/文案迭代优化。
- OpenClaw + autoresearch 工具链 — 自动化实验/搜索/优化闭环，能在营销、投放、转化等场景中持续逼近最优解。
- Lynx / Vue Lynx — 跨平台 UI 引擎示例，展示用 AI 快速完成跨框架集成与工程化迁移的实践模板。

**核心观点/方法论**（如有）
- AI 是工具不是魔法 — 重要在于设定正确期望、把 AI 当成能放大人类能力的执行器而非全能替代品。
- 提问与批判性思维决定产出质量 — 好的问题和 prompt 比盲目依赖模型更关键，会直接影响效果与效率。
- Agent 友好优先（Agent-friendly） — 让运行环境、路径、工具对 Agent 可观察、可修改，能显著降低调试和自动修复成本。
- 投资 AI 做不到的事 & 不过度依赖 — 保持对不可自动化能力的投入（创意、战略、人际）能形成长期竞争力。
- 从“推理思维”到“智能体思维” — 更强调为了行动而思考：边想边做、工具调用与环境反馈的闭环是未来核心能力。

**实践经验/案例**（如有）
- 给高中生演示 AI 编程：做小游戏/小工具最有效，易理解且能激发创造力与实操动手能力。
- bun + 相对路径 + baseDir 策略：在 skills 中用 bun 执行 TypeScript 并告诉 agent 基础目录，能显著降低路径错误与打补丁难度。
- Type4Me 案例：本地优先的语音识别 + LLM 后处理和命令模式，示范了隐私可控且可扩展的产品化路径。
- Autoresearch/OpenClaw 成功样例：在营销分人群、测试素材、调参等任务上实现持续自动化优化，减少人工盯盘。
- 黄玄的 Vue Lynx 实践：用 AI 保持跨 session 架构一致性、桥接测试套件与自动化验证，展示 AI 驱动开发的工程化方法论。
