**新工具/技术**
- OpenAI Codex 插件系统：把技能提示词（Skills）、应用集成（Apps）和 MCP 配置打包成 plugin.json + 技能目录，支持本地安装、CLI 加载、版本管理和市场分发，便于团队一键共享开发工作流。
- OpenAI 官方开源插件：官方已开源多款插件，提供可复用的脚手架和示例，加速上手与二次开发。
- Google Translate 耳机实时翻译：基于 Gemini 的实时语音翻译，支持任意带麦耳机、70+ 语言并保留语调，降低硬件门槛提升实时对话体验。
- Anthropic 新模型（Claude Mythos / Capybara）：内部定位为比 Opus 更强的新一代大模型，强调在编程、推理与网络安全能力上的显著跃升。

**核心观点/方法论**
- 插件=工作流打包：把技能、应用与运行配置捆绑能标准化团队内的工程/产品流程，降低复制成本并提升协作效率；对大型模型工程化交付非常重要。
- 开源与目录化推动生态：官方开源 + 精选目录能快速降低接入门槛，形成网络效应，加速插件质量迭代与社区贡献。
- 安全优先的分阶段发布：Anthropic 选择先向防守方开放以缓解可能的攻防风险，体现高能力模型应优先考虑滥用风险与防御方能力平衡。
- 以财务与合规驱动产品取舍：OpenAI 砍掉多条亏损或高风险产品，显示在 IPO 前需清理潜在头条风险并集中资源于可变现的“超应用”路线。
- 降低使用门槛即增长杠杆：Google 取消硬件限制说明减少生态摩擦（硬件/安装）比单纯功能更能快速扩大用户层。

**实践经验/案例**
- 插件实现要点：用 plugin.json 描述清单、按技能目录组织能力、提供本地/CLI 安装与版本管理，便于 CI/CD 与团队复用。
- 开源插件实践：官方样例能作为模板，团队可直接 adapt 并上本地市场，加速落地与内部标准化。
- 配置管理教训（Anthropic 泄露）：CMS 配置错误导致草稿公开，提醒必须把敏感草稿与发布流程做严格权限与审计控制。
- 产品关停案例启示：Sora 因运行成本远高于收入被砍；Instant Checkout 转化率低、税务与库存同步欠缺；成人聊天因年龄验证与合规问题被无限期搁置，说明要在早期就量化商业可行性与合规风险。
- 推广策略：Codex 重置额度以鼓励开发者试用插件，是通过放宽门槛刺激生态活跃度的有效短期策略。
