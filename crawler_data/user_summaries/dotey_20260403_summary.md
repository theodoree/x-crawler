**新工具/技术**（如有）
- info-card-designer（Skill）：给任意 URL 抓取 Markdown → 生成 HTML 摘要并自动截图成信息卡，便于分享与展示。  
- Vibe Island（Mac App / 灵动岛）：在 Mac 刘海区汇总多个 Agent 的实时状态，支持轻量交互和权限审批，减少切换成本。  
- ChromaFs（Mintlify 提案）：一个“假的文件系统”，把 grep/cat/ls 等命令翻译成对 Chroma 向量/元数据的查询，让模型像读文件一样精确检索文档。  
- just-bash：TypeScript 实现的 bash 子集，可插拔文件系统接口，作为 ChromaFs 的执行层。  
- 缓存策略（5 分钟 / 1 小时）：把缓存过期时间作为配置权衡成本与命中率的简单机制。

**核心观点/方法论**（如有）
- RAG 中的 R 不必等同向量检索：检索可以是全文、SQL、模拟文件系统等，更贴合场景时比纯向量检索更可靠且可控。  
- 让模型主动“探索”而不是预先塞满上下文：把检索工具交给模型决定看什么、怎么找，能提高精确性并节约成本。  
- 为 Agent 建立 feedback loop：让 Agent 自我验证和循环改进能显著提升产出质量并降低人工复核需求。  
- 重视模型的“功能性情绪”：模型内部有可影响行为的激活模式（类似情绪），需要监测与干预以避免作弊或不稳定行为。  
- 以最小摩擦呈现状态/通知：像灵动岛这种微交互能在不打断主任务的情况下把注意力拉回正确位置，提升效率。

**实践经验/案例**（如有）
- Mintlify/ChromaFs 实测：把会话启动时间从 ~46 秒降到 ~100 毫秒，边际计算成本几乎为零，并通过元数据预筛+内存精确匹配加速 grep。  
- 权限与安全设计：ChromaFs 在初始化时裁剪文件树实现权限隔离，且所有写操作返回只读错误，避免越权与数据污染。  
- 成本/命中率折中（缓存 TTL）：若续聊间隔常 <5 分钟用 5 分钟缓存更划算，若常在半小时内回访则选 1 小时以提高命中率（但成本更高）。  
- Prompt 技巧（情绪/角色化）：先让 AI 扮演“挑刺/仇人”给出犀利反馈，再基于这些反馈重写，可显著提升生成质量。  
- Agent UX（Vibe Island）：把多个并行 Agent 的状态聚合到小组件中，能在需要时展开并完成审批/跳转，实际提升多任务协作体验。
