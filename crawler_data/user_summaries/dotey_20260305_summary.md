**新工具/技术**  
- Google Workspace CLI — Google 为 Workspace 推出了命令行工具，便于脚本化和自动化办公流程，适合工程化运维与批量处理。  
- GPT-5.4（OpenAI）— 原生支持“电脑操控”（截图、点击、键盘输入）、按需 Tool Search（减少 token 消耗 ~47%），在桌面操控和专业知识型任务上显著提升。  
- Opus 4.6 — 在多稿合并/文本合并场景中被作者推荐，当前效果最好（用于生成/合并文本草稿的模型）。  
- 自动配图工具/流程 — 自动为文章配图的能力被多次提及，成为写作到发布的自动化环节之一。  
- dotey 的开源 skills 仓库 — 作者发布的技能集，用于把写作/发布流程模块化、自动化（方便复用和工程化）。

**核心观点/方法论**  
- AI 是杠杆，适用于程序员和人文工作者 — 会把重复性工作交给模型，释放人去做更高维的判断与创作；这对效率和产出质控都很重要。  
- 写作可以“工业化”——把写作拆成素材采集、提炼、生成大纲、多稿生成、合并、配图、发布等阶段，用不同模型/流程各司其职，利于规模化生产。  
- 人机协同需人类审美把关 — AI 在审美/品味上不足，人工介入（指定基准稿、审校局部）能避免“让AI自己决定风格”的退化循环。  
- 既有产品改造与重做的权衡 — 对于体量大的产品，直接做 AI 原生的新产品在很多场景下更可行，但要考虑数据/经验的保留价值。

**实践经验/案例**  
- 多稿合并实践：最好允许人工指定“以哪个稿子为基础”并明确其它稿的合并规则，效果优于让 AI 全权决定。  
- 推荐使用 Opus 4.6 做合并任务，作者实测认为效果最好（实际环境需自行验证）。  
- 作者的写作流水线示例：用 AI 分析提炼素材 → 生成多份大纲/草稿 → 合并润色 → 自动配图 → 自动发布，提升产量但受限于审美高度。  
- 人才流动与生态影响：Qwen 团队离职引发市场担忧（阿里股价波动），显示核心人才流动会影响开源/前沿模型的市场信心。  
- GPT-5.4 的实战意义：桌面操控和按需工具检索可把许多“需要外部交互/工具”的流程自动化（节省 token、提高成功率），但 API 价格和产品集成成本也上升。  
- Agent 化趋势：Manus 一年成长与用户案例展示了“Agent+人”的落地价值，说明 agent 工具在可用性和生产力上快速成熟。
