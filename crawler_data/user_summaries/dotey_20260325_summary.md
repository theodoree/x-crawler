**新工具/技术**
- Skills（泛指） — 可封装人为决策、反馈或处理流程的模块化能力，Agent 可自主调用，既可作为后台能力也可当作 slash command 暴露给用户。
- Slash command — 用户主动触发的命令接口（例如 /init），适合交互启动或单次操作，与 Skills 在触发方式上互补。
- OpenCLI（1.4/1.5 路线） — 命令行工具集，新增反风控、插件化、平台适配与 pipeline 引擎等，目标提升易用性与安全性。
- bb-browser（badboy browser） — 通过 Chrome 插件 + CDP 在真实浏览器中直接执行脚本，利用用户登录态爬取和操作网站，获取信息更稳但有安全/合规风险。
- ljg 系列 Skills（ljg-card / ljg-learn / ljg-paper / ljg-travel 等） — 面向内容生产与知识工作的一套技能集，支持从论文阅读到视觉卡片、旅行研究的一键化流程。
- Slides skill（AJ 的 skill） — 用 skill + nano banana API 生成高质量幻灯片，适合快速把分享内容自动制图出图。

**核心观点/方法论**
- Agent + Skills 能把重复性判断/反馈自动化 — 对于经常重复相同反馈的管理者，构建“exec-review”类 skill 能显著放大时间效率并保持风格一致。
- 触发与治理要分层：slash command 适合显性用户驱动，Skills 适合隐性/自动化场景，两者可以互相映射与互用，设计时注意授权和可控性。
- 插件化与反风控是可扩展系统的关键 — OpenCLI 的方向说明在易用性提升之外，必须考虑 anti-detection 与安全加固，避免被平台机制或攻击绕过。
- 用真实浏览器操控可解决鉴权与复杂前端的问题 — bb-browser 的思路用得“方便且强大”，但同时带来隐私、合规与安全边界的挑战。
- 工作流（workflow）将多个 skills 串联成端到端命令，是把点状能力转为可重复产出的关键工程实践。

**实践经验/案例**
- VP 用例：有人把自己给团队的反馈封装成 skill（exec-review），团队效率大幅提升，说明中层知识/风格可工程化复制。
- OpenCLI 与社区协作：dotey 对 OpenCLI 的建议被采纳，项目在反风控、安全和插件化上有大量实装与迭代，可作为命令行 Agent 化的参考。
- bb-browser 实测反馈：用户试用后评价“很好很强大”，验证了通过浏览器真实会话做 agent 化抓取/自动化的可行性，但需权衡风险与合规控制。
