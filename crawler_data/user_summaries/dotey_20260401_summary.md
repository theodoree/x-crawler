**新工具/技术**（如有）
- SentrySearch：用多模态嵌入实现按语义检索视频的开源命令行工具，直接把视频分块编码进向量库并裁剪命中片段。
- Gemini Embedding 2 / Qwen3‑VL‑Embedding：多模态嵌入模型，支持把视频像素与文本映射到同一向量空间，避免先转录为文字的开销。
- ChromaDB：本地向量数据库，用于存储视频片段向量，支持离线检索与导出片段。
- Claude Code 与相关工具（claude-trace、NO_FLICKER）：源码泄露引出对 Agent Loop、记忆管理和终端渲染新模式的可视化与调试工具链讨论。

**核心观点/方法论**（如有）
- 学习开源项目的步骤：先跑起来 → 以点带面分析功能 → 二次开发定制 → 最后从零重构超越，循序渐进能最快掌握架构。
- 直接在向量层面比较像素与文本（多模态嵌入）是大规模视频语义搜索的可行路径，降低成本并提高检索语义性。
- 开源 vs 闭源是权衡：闭源便于快速迭代、隐藏实现细节与防蒸馏，但开源有利于社区审查和生态建设。
- 记忆系统应包含情景/语义/程序化三层：不仅存历史，还要蒸馏知识并记录可复用的技能（正向与负向反馈均重要）。
- 工程上要把注意力放在流程与自动化上：把事故归因于流程/infra 而非个人，优先改进自动部署和 CI。

**实践经验/案例**（如有）
- SentrySearch 成本与部署：云端一小时视频索引约 2.84 美元；完全离线可跑本地 Qwen3‑VL（需 ~24GB+ 显存/内存 的机器）。
- 视频分块带重叠是为避免把目标事件切在片段边界，重叠与窗口大小需在覆盖率和索引量间权衡。
- 分析泄露代码实操：先把系统跑起来，打印或拦截所有 API 请求（如用 claude-trace），可直观重建 Agent Loop 和 prompt 流向。
- Claude Code 的记忆实现（jsonl 存储、SessionMemory、extractMemories、autoDream）是典型的工程化记忆压缩与巩固流程，但在语义化召回和长期存储策略上仍有改进空间。
- NO_FLICKER 终端模式示例：用备用屏幕缓冲区消除闪烁并支持鼠标，但会改变原生搜索/剪贴板交互，需权衡可用性与体验。
