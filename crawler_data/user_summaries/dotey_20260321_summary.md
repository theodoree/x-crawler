**新工具/技术**
- 高质量合成数据：用于 LLM 训练，目标是替代人工标注——成本更低、速度更快、错误率更小，适合大规模训练/微调场景。  
- 多模态检索/搜索：被提为下一波主流入口，尤其在视频/图像+文本场景下对检索与代理能力提升显著。  
- Claude Code（及相关设置技巧）：Claude Code 会自动把自己加为 git co-author，实用技巧是在 ~/.claude/settings.json 里清空 attribution 字段以默认不加 co-author。  
- 视频检索/索引与嵌入技术：Panda 在做的视频检索/embeddings 产品方向，强调为 AI agent 提供高质量视频索引能力。

**核心观点/方法论**
- 合成数据会替代人工标注：理由是成本、速度和质量优势，长期会改变数据供给与标注市场格局。  
- 创业与融资要有坚定判断：早期不够坚定可能错失扩张和机构背书的机会，快速融资+认真做 12 个月能放大选择（继续做或卖出）。  
- 代码/文档组织以“可进单次上下文”的自洽单元为目标：把复杂架构按功能拆文件，让 AI 能根据文件名索引并在一次上下文内理解，稳定性高于让模型到处拼凑猜测。  
- 技术写作/知识沉淀很重要：团队核心成员（如 Claude Code 的作者）把经验写成串能比商业课程更有价值，便于实践传播。

**实践经验/案例**
- 融资复盘（Panda）：24 年底已拿到约 150 万美元的真实数据订单，但因为不够坚定只跟少数机构聊，认为若当时广撒网可在首轮拿 200–300 万美元并通过机构背书实现更好增长或退出。  
- 文件拆分实操：对于有复用或复杂状态流转的逻辑必须拆；如果没有共享逻辑或状态机，文件膨胀未必引入维护成本，但为了让 AI 协同更稳仍建议按可装进单次上下文拆分并定期让 AI 重构。  
- 小工程技巧：在 ~/.claude/settings.json 加入 {"attribution":{"commit":"","pr":""}} 可默认不把 Claude 标为 co-author，解决代码仓库里自动署名的问题。
